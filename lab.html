Security & Systems Lab

The current CS job market puts a lot of weight on real, hands-on experience, especially for security and infrastructure roles. This lab is my way of building that experience: a controlled environment where I can practice SOC workflows, log analysis, and incident response without touching production systems.

Current & Planned Environment

  • Virtualization: Host machine running multiple VMs (Linux and Windows).
  • Operating systems: Ubuntu Server, a Windows workstation/server VM.
  • Security tools: Splunk (or similar SIEM), basic firewall configurations, and log generators.
  • Networking: Simulated internal network traffic, SSH/RDP access, and basic hardening.

The goal is not just to “install tools,” but to understand how logs flow, how to build useful dashboards and searches, and how alerts connect to real-world behaviors.

What I Practice in the Lab

  • Log ingestion & normalization: getting logs from multiple systems into a single place and making them queryable.
  • Detection strategies: building searches or alerts for suspicious patterns (failed logins, privilege changes, odd network behavior).
  • Investigation workflow: following an alert from initial signal to root cause using available data.
  • Reporting: writing small incident summaries that would make sense to a SOC lead or manager.

Roadmap

  • In progress Set up a base lab with Linux and Windows VMs and central logging.
  • Planned Integrate a SIEM tool (Splunk, Graylog, or similar) and forward OS logs and application logs.
  • Planned Simulate common attack patterns (brute force, suspicious processes, simple malware) in a safe environment.
  • Planned Create a small “lab playbook” documenting detection, investigation, and reporting steps for each scenario.

Over time, I plan to package parts of this lab into a public guide and/or GitHub repository so other students can replicate it and practice similar skills.
